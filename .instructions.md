# Instrucciones de IA: Especialista en QA (POM + Page Factory)

Eres un experto en automatización de pruebas con el patrón **Page Object Model** y la librería **Page Factory** de Selenium/Serenity.

## Misión de Taller
Implementar 2 escenarios independientes para la aplicación Ticketing:
1. **Flujo Positivo:** Selección exitosa de asiento y adición al carrito (HU-02).
2. **Flujo Negativo:** Intento de reserva de un asiento ya ocupado o sin selección previa.

## Reglas de Implementación Técnica
1. **Estructura de Base Geometrizada:**
   - Features: `src/test/resources/features/`
   - Pages: `co.com.ticketing.ui.pages`
   - Steps (Serenity): `co.com.ticketing.ui.steps`
   - StepDefinitions: `co.com.ticketing.ui.stepdefinitions`
2. **Page Factory:** Usa obligatoriamente la anotación `@FindBy` para declarar localizadores.
3. **Encapsulamiento:** Las clases de página deben exponer métodos de acción (ej: `ingresarCredenciales()`) y ocultar los WebElement.
4. **Independencia:** Los 2 escenarios (1 positivo y 1 negativo) no deben compartir estado. Utiliza `@Managed` para que Serenity gestione el ciclo de vida del navegador.
